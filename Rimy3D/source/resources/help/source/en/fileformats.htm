Supported file formats

Apart from various game files formats, Rimy3D also supports several universal 3D file formats like 3DS, OBJ or ASE.
However, each of these file formats has some shortcomings, for example an object's position cannot be saved in the OBJ format.

To solve this problem Rimy3D implements its own file format with the ending .3db that can save all of the typically needed information. Through the 'Tools' menu in Rimy3D you can install import/export plug-ins for the 3D editors GMax, Blender and 3ds Max.
Whenever possible you should prefer 3DB over the other file formats!

Rimy3D is capable of automatically finding the necessary materials and textures when loading 3D files even if they are not in the same folder. For this to work you need to configure the search paths.

Below are lists of the supported 3D, image and material file formats.
